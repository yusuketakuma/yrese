## 1. 目的と結論

イベントストア(ARC-005)から導出される投影(read model)と、算定のやり直し(再算定)の境界を確定する。

**結論: 投影は使い捨て可能な導出物であり、いつでもイベントから再構築できる。ただし「確定済み請求」はいかなる再投影・再算定でも書き換えない — 訂正は常に新しいイベントの追加である。**

## 2. 投影の規律

1. **イベントが正本、投影は導出物**。投影の破棄・再構築は業務データの損失ではない(バックアップ対象の中心はイベントストア)。
2. **投影の再構築は決定的**。同じイベント列から同じ投影が得られる(CAL-010 の決定論規律を投影にも適用。現在時刻・現行マスターへの暗黙参照禁止)。
3. **投影はバグ修正で自由に直せる**。投影ロジックのバグは投影の再構築で解消する — これは silent fix ではない(イベント=事実は変わっていないため)。
4. 投影の再構築中もイベント追記は止めない(24/365 — WP-0045。具体的な無停止再構築方式は同 SSOT で確定)。

## 3. 再算定の規律

1. **再算定 = 新しい算定イベントの追記**。過去の算定結果(および calculation_trace)は訂正後も削除・上書きしない。
2. 再算定は**当時版の再現**(旧ルールセット版・旧マスター版での再実行 — 監査・返戻対応)と**新版での算定し直し**(訂正・改定対応)を区別し、どちらの実行かをイベントに記録する。どちらも CAL-009 の版指定で決定的に実行できること。
3. **確定済み請求に対する再算定は、確定を自動では変更しない**。新しい算定結果が確定済み請求と異なる場合、それは ARC-007 の訂正レーン(返戻・再請求等)の起点となる業務イベントであり、自動上書き・自動再提出は禁止する。
4. 未確定(下書き・仮算定)の算定結果は再算定で置き換えてよい(ただし trace は残す)。

## 4. 集計との関係

- QUA-009(返戻率KPI)等の集計は**確定イベントのみ**を分母分子にする(fail-closed 集計)。投影の再構築や未確定データの再算定は集計値を変えない。
- 集計値の遡及訂正が必要な場合(確定イベント自体の訂正レーン発動時)は、訂正履歴付きで新しい集計版を発行する(QUA-009 §fail-closed 集計 5 規律)。

## 5. 停止条件(fail-closed)

- 再投影による確定済み請求・確定イベントの書き換え → 実装禁止(ARC-007)
- 非決定的な投影(現在時刻・現行マスター参照) → CHANGES_REQUESTED
- 再算定結果による確定請求の自動上書き・自動再提出 → BLOCKED(訂正レーンへ差し戻し)
- イベントを経由しない投影への直接書き込み → 実装禁止
