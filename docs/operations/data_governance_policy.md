## 1. データ所有権・責任分界

- **薬局データの所有権は薬局(テナント)にある**。当社は医療情報を取り扱う情報システム・サービス提供事業者(SEC-002)として受託処理する
- 管理者(薬局)/処理者(当社)の責任分界を契約(データ処理契約)で明文化する【要確認 — 法務】
- 薬局間・法人間のデータ混在は重大事故として扱う(SEC-006 のテナント分離が技術統制)

## 2. データ分類と保持

| 分類 | 例 | 保持期間 | 根拠 |
|---|---|---|---|
| 法令保存データ | 調剤録・レセプト・帳票原本・算定根拠 | 法定期間【要確認 — REG-003】+テナント指定延長 | 薬剤師法等 |
| 監査証跡 | 監査ログ・移行ログ・サポートアクセスログ | 法定+運用要件(候補: 5年以上)【要確認】 | SEC-007 |
| 運用ログ | デバッグ・メトリクス | 短期(候補: 90日〜1年) | OPS-009 |
| バックアップ | DB スナップショット・Edge ローカル | 保持世代・期間を定義(候補: PITR 35日+月次1年)【候補値】 | OPS-005 RPO |

- 削除と法令保存義務の衝突は未整理のまま運用しない(v0.2.0 §9.5 禁止)— 削除要求時は保存義務データを除外し、その旨を要求者へ明示する
- legal hold: 監査・訴訟・行政調査時は対象データの削除・変更を凍結する(発動権限者【要確認】、発動・解除は監査ログ)

## 3. ライフサイクル統制

- 生成: PHI classification(@yrese/events)を生成時に付与
- 利用: 目的外利用禁止(SEC-004 の処理目的台帳に限定)
- 提供: Pharmacy Integration API 経由の第三者提供は薬局の明示設定+scope 制御+監査ログ
- 保管: 暗号化(at rest/in transit)、鍵分離(SEC-006)
- 廃棄: データ削除・媒体廃棄は廃棄証跡(何を・いつ・誰が・方法)を残す。バックアップ内残存の削除反映期限を定義(候補: 世代満了時)【候補値】

## 4. テナントイベント時の扱い

| イベント | 扱い |
|---|---|
| 薬局閉局 | 法令保存データの返却+保存義務継続分の取り扱い合意(OPS-011) |
| 法人変更(承継) | 承継先への移転は法的根拠+旧法人の同意記録を確認【要確認 — 法務】 |
| 薬局譲渡 | 同上。tenant_id 付け替えではなく移転手続きとして監査ログ化 |
| 解約(offboarding) | OPS-011 のエクスポート→確認→削除フロー |
| サービス終了 | OPS-011 の移行支援+返却+削除証跡 |

## 5. 禁止事項(v0.2.0 §9.5)

- 解約時に薬局が必要な法令保存データへアクセス不能になること
- エクスポートデータから算定根拠・マスター版・帳票版が失われること
- 監査ログを通常データと同じ権限で削除できること(SEC-007 の権限分離)
- データ削除と法令保存義務の衝突を未整理にすること
