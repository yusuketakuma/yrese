## 1. 並行稼働方式

| 方式 | 内容 | 適用 |
|---|---|---|
| 全件並行 | 全処方を新旧両システムに入力し全件照合 | 高リスク薬局・初期導入店舗(推奨) |
| 標本並行 | 代表的な処方パターン(既知処方案件)を新系で再現し照合 | 全件並行の負荷が許容不能な場合(人間レビューで承認) |

照合対象(v0.1.7 §9.7):

- 既知処方案件での算定照合(点数・患者負担・保険請求・公費請求)
- 既知請求案件でのレセプト照合(記録内容の一致)
- 帳票照合(領収証・調剤明細書・調剤録)
- 会計照合(未収・返金・差額精算)
- 返戻・再請求シナリオ確認

差分の扱い:

- 全差分を「新系起因 / 旧系起因 / マスター版差 / 運用差」に分類し台帳化
- 新系起因の算定差分は severity high 以上の欠陥として扱う(CAL-001 の根拠と突合)
- 許容差分の定量基準は Go/No-Go(OPS-012)の入力とする【要確認】

## 2. カットオーバー手順(概要)

1. Go/No-Go 判定(OPS-012)通過
2. 旧システム入力凍結ポイント宣言(請求月境界)
3. 最終差分移行(凍結後の増分)+ 照合
4. 新システム本番モード開始(切替日時を監査ログへ記録)
5. 旧システムは参照専用モードへ(参照期間【要確認】)
6. 切替後 2 週間は強化監視(OPS-009 のダッシュボード+日次差分レビュー)

## 3. ロールバック(戻し)手順

- 発動条件: 切替後に請求不能・算定重大差分・データ破損級の障害が発生し、当該請求月内に修正見込みが立たない場合
- 手順: (1) 新系入力凍結 → (2) 新系で入力済みデータのエクスポート(evidence_id・マスター版込み)→ (3) 旧システム入力再開 → (4) 新系入力分の旧系再入力(照合付き)→ (5) インシデント報告(OPS-013)
- 戻し手順がリハーサル未実施のままカットオーバーすることを禁止(BLOCKED_CUTOVER_ROLLBACK_UNDEFINED)
- rollback rehearsal は release_gate_policy(WP-0011)のリリース前必須項目と同期

## 4. 禁止事項

- Go/No-Go 未通過でのカットオーバー
- 並行稼働差分の未分類放置(全件分類が完了するまで判定に進まない)
- 切替日と請求月境界の不一致(例外は人間レビュー)
- 旧システムの即時廃棄(参照期間・保存義務の確認前)
