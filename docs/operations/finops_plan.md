## 1. コスト構造(薬局あたり月額の内訳見積り枠)

| 費目 | 主要ドライバ | 見積り方法 |
|---|---|---|
| コンピューティング(ECS Fargate) | テナント数・同期/バッチ流量 | OPS-006 負荷モデルから単価計算(Phase 1 実測) |
| DB(Aurora) | レセプト件数・監査ログ・イベントストア | ストレージ増分×保持期間(OPS-010) |
| ストレージ(S3) | 帳票 PDF・電子レセプト・バックアップ・エクスポート | 件数×平均サイズ×保持 |
| ログ・監査証跡保管 | OPS-009 ログ量+SEC-007 監査ログ | 階層化(ホット→コールド)で最適化 |
| バックアップ | PITR+スナップショット世代 | OPS-005 RPO 設定から算出 |
| 外部接続 | IP-VPN/IPsec 等の請求系回線【要確認】・VPC endpoint | 公式接続方式確定後(REG-007) |
| Edge Node | ハード原価・更新配布帯域・予備機 | 提供形態【要確認】 |
| サポート | 問い合わせ件数×処理時間(OPS-004) | 導入数に比例+請求期ピーク |
| 帳票・印刷関連 | 用紙・プリンタ保守は薬局側(契約で明確化) | — |

## 2. コスト監視・異常検知

- AWS コスト異常検知(Cost Anomaly Detection)+予算アラート(月次予算の 80%/100%/120%)
- テナント別コスト配賦: tenant_id タグでリソース・ログ量を按分し、薬局あたりコストを月次算出
- コストダッシュボードを月次経営レビューへ(OPS-009 とは別系統)
- 増分コストレビュー: 新機能の設計時にコスト影響欄を PR テンプレートで必須化(特にログ・イベント・帳票生成)

## 3. 最適化方針(許可される節約)

- ログ・帳票の階層化ストレージ(アクセス頻度で移行)
- 非本番環境の自動停止
- 同期イベントのバッチ化・圧縮(正確性・監査性を損なわない範囲)
- Savings Plans / Reserved は負荷実測後(Phase 2 以降)

## 4. 禁止事項(v0.1.7 §9.8 — 絶対条件)

- SLA/SLO 未定義で本番提供すること
- **コスト削減のために監査ログ・バックアップ・暗号化を削ること**
- 契約終了時のデータ返却・削除方針を未定義にすること(OPS-011 で定義済み)

## 5. 契約・責任分界(§9.8 整理対象)

- メンテナンスウィンドウ・障害通知・インシデント通知: OPS-005 §4 / OPS-004 §4

- サポート範囲・対象外条件: OPS-004 / OPS-007(デバイス対象外条件)
- サービス終了時の移行支援・データ返却・削除: OPS-011
