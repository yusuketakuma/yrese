## 1. エクスポート対象(v0.1.7 §9.5 の全項目)

| # | 対象 | 形式(候補) | 必須付帯情報 |
|---|---|---|---|
| 1 | 患者情報 | CSV+JSON | 旧患者ID対応表 |
| 2 | 保険・公費情報 | 同上 | 有効期間・資格確認履歴 |
| 3 | 処方・調剤履歴 | 同上 | 医薬品コード(複数体系 — MST-002 対応表付き) |
| 4 | 請求履歴 | 同上+電子レセプトファイル | 請求月・返戻・再請求状態 |
| 5 | 帳票 | PDF(出力時点の版) | 出力履歴・ハッシュ |
| 6 | 監査ログ | JSON Lines | ハッシュチェーン検証情報(SEC-007) |
| 7 | マスター版情報 | JSON | 各データが依拠したマスター版・算定ルール版 |
| 8 | evidence_id / legal_trace / calculation_trace | JSON | 算定根拠の追跡可能性を維持 |

原則: **エクスポートデータから算定根拠・マスター版・帳票版が失われてはならない**(v0.1.7 §9.5 禁止)。データ辞書(スキーマ定義書)を添付し、第三者が解釈可能にする。

## 2. 提供経路

- セルフサービス: 管理者権限(tenant admin)でのエクスポート要求 → 非同期生成 → 暗号化ダウンロード(有効期限付き)+ 監査ログ
- API: Pharmacy Integration API の data export scope(大容量は非同期ジョブ)
- 解約時: offboarding フローに組み込み(下記)

## 3. Offboarding フロー(解約・閉局・移行)

1. 解約申請の受理(本人性確認+権限確認)
2. エクスポート生成(全対象)+ データ辞書
3. 薬局側の受領確認(内容検証期間: 候補 30日)
4. 法令保存義務データの取り扱い合意(薬局側保管 or 保管継続契約)【要確認 — 法務】
5. 削除実行(バックアップ含む反映期限は OPS-010 §3)+ 廃棄証跡発行
6. 削除完了通知+証跡の交付

- 受領確認前の削除実行を禁止する
- 解約後も法令保存データへのアクセス手段が失われないこと(v0.1.7 §9.5 禁止に対応)

## 4. ベンダーロックイン低減

- コードは公的コード体系(レセ電算・YJ・一般名等)を正とし、独自コードには必ず公的対応表(MST-002)を付ける
- 帳票・レセプトは公式形式(記録条件仕様)で再出力可能に保つ
- API のデータ取得範囲はエクスポートと同等以上を目標(段階整備)
- サービス終了時: 6ヶ月以上前【候補値】の告知+移行支援+全テナント返却完了までの運転継続を計画
