## 0. 位置づけと境界(必読)

- **医療保険(調剤報酬点数表 S1)とは別系統。** CAL-003 evidence_register は医療保険のみを台帳化しており、
  介護保険の居宅療養管理指導費(薬局の薬剤師が行うもの)は **evidence 未採番**。医療保険側の点数を
  介護保険項目へ流用してはならない(単位=点数ではない。1単位の金額・地域区分単価も別体系)。
- **非MVP。** non_mvp_scope N4「在宅関連の複雑算定・介護保険請求」/ facility_billing_policy「介護保険連携は
  非MVP: PRD-002」。現行の製品挙動は在宅・介護系入力を `BLOCKED_UNSUPPORTED_CLAIM` とする(N4 の対処)。
  本台帳は**その解除に先立つ準備作業**であり、evidence 発行=即実装可(APPROVED_FOR_IMPLEMENTATION)ではない。
- **捏造禁止。** 単位数・算定回数上限・加算の要件・地域区分単価は、原本を判読(CONFIRMED_VISUAL)し
  evidence_id を発行するまで本書に数値を書かない。項目名称は令和6年度介護報酬改定時点の一般的呼称で
  仮置きし、すべて【要改定確認】とする。

## 2. 単位→金額の換算体系(医療保険と異なる論点)

- 介護保険は「単位数 × 地域区分別1単位単価(円)」で費用額を算出し、給付率・利用者負担が別途かかる。
- 1単位単価は**サービス種別×地域区分**で異なり、告示別表で定義される。MOD-010 §1(IEEE-754 禁止・
  bigint/ScaledDecimal のみ)と丸め規律(公式根拠 evidence 必須)は医療保険と同様に適用する。
- 換算・端数処理の細部は原本(単価告示・介護給付費請求に関する省令/通知)で確定するまで実装しない。

## 4. 正式 SSOT 化の手順(graduation)

CAL-001/CAL-003 の解除手順に準拠する:

1. S-K1〜S-K5 の版・適用日・入手元を **source_registry(REG-007 系)へ登録**。
2. 判読(CONFIRMED_VISUAL)し、**新 SSOT `docs/calculation/kaigo_kyotaku_ryoyo_evidence_register.md`
   (ssot_id 案: CAL-012)** を新設。frontmatter(ssot_id/status/owner/reviewers 等)を CAL-003 準拠で作成し、
   **`docs/ssot_index.md` の docs/calculation/ 件数・総文書数・行を整合更新**(このSSOT新設は index 改版を伴う)。
3. evidence_id(例: `EVD-KKR-0001`…)を発行し、単位数・回数上限・加算要件を行単位で追記。
4. golden test 期待値を原本から作成し、medical_safety_reviewer / claims_evidence_specialist ほかの
   レビューパイプラインで APPROVED を得る(SaMD該当性・請求正当性の観点含む)。
5. 実装解禁は **非MVP N4 の製品判断(PRD-002)** と整合してから。それまでは `BLOCKED_UNSUPPORTED_CLAIM` を維持。
