## 2. 通信・ファイル形式(本文 p.1、直接確認)

- 電気通信回線: ISDNダイヤルアップ / 閉域IP網の IP-VPN / オープンネットワークでは IPsec(IETF標準)+ IKE
- 記録形式: **CSV形式**
- ファイル名: **"RECEIPTY"**、拡張子: **"CYO"**
- 1ファイル=1ボリューム。収まらない場合はレセプト単位に分割し別ボリューム(マルチボリューム識別情報 "00" から昇順2桁、レセプトがボリュームをまたぐ分割は禁止)
- レコードは改行コードで区切り、最終レコード後に EOF コード

## 3. レコード種別(本文 p.1-2 の表から転記)

### 3.1 ファイル構成(1ファイル内・記録順)

|---|---|---|---|

### 3.4 レセプト種別ごとの記録条件(本文 p.2 の表、要点)

- 医療保険単独: HO=○、KO=×、SN/JD=△(1レコード)、MF=△
- 医療保険+公費1〜4種併用: HO=○、KO=○(1〜4)、SN=△(1〜5)、JD=△(2〜5)、MF=△
- 公費単独: HO=×、KO=○(1)、SN/JD=△(1)、MF=△
- 公費2〜4併用: HO=×、KO=○(2〜4)、SN=△(1〜4)、JD=△(2〜4)、MF=△
- 公費レコード複数時は「法別番号及び制度の略称表」(記載要領)の順で第一公費→第四公費の順に記録

## 4. レコード形式・文字符号(本文 p.3、直接確認)

- 可変長レコード、各レコード末尾に改行コード。項目間はコンマ区切り(数値の位取りコンマ使用禁止)
- モード: 数字(半角数字)/英数(半角英数)/漢字(全角)。可変項目は有効桁まで(ゼロサプレス。ただし別表等規定コードの先頭 "0" は保持 — 例: コード "01" は "01" のまま)
- 年は西暦。年月 "YYYYMM"(6桁)、年月日 "YYYYMMDD"(8桁)、時刻 "hhmm" 24時間表記
- 文字符号: **JIS X 0201-1976 8単位符号 + JIS X 0208-1983 附属書1 シフト符号化表現(シフトJIS)**
- 制御符号: コンマ (2C) 1バイト=項目区切り / 引用符 (22) 使用しない / 改行 (0D)(0A) 2バイト=レコード区切り / EOF (1A) 1バイト
