## 1. 原則(v0.1.7 §21)

- 自動更新であっても、**請求・算定に影響するマスターは即時本番反映しない**。
- 失敗時は `PENDING_MASTER_VALIDATION`(@yrese/shared-kernel 実装済み)とし、**旧版マスターで業務継続**する。
- 処方日・調剤日・請求月に応じて適切なマスター版を選択する(有効開始日・廃止日・経過措置)。
- マスターは有効日・廃止日・経過措置を必ず持つ(v0.1.7 §3)。

## 2. 対象マスター

医薬品(薬価)/ 調剤行為 / コメント / 保険者 / 公費 / PMH制度関連 / 医療機関 / 用法コード / 電子処方箋医薬品コード対応表(各配布元・形式は source_registry で管理。全て【要確認】)

## 3. 24段パイプライン

| # | 段階 | 検証・処理内容 | 失敗時挙動 |
|---|---|---|---|
| 1 | 取得 | 配布元からのダウンロード(取得日時・取得元URL・担当を記録) | リトライ後 PENDING_MASTER_VALIDATION、旧版継続 |
| 2 | 署名/ハッシュ確認 | 配布元提供の署名・ハッシュ検証(提供有無【要確認】。なければ取得時ハッシュを自己記録) | 不一致は即中止+セキュリティ通報 |
| 3 | ファイル形式検証 | 拡張子・レイアウト・レコード種別 | 中止・保留 |
| 4 | 文字コード検証 | Shift-JIS/UTF-8 等の宣言と実体の一致 | 中止・保留 |
| 5 | スキーマ検証 | 列数・型・必須項目 | 中止・保留 |
| 6 | 差分検出 | 現行版との追加/変更/廃止差分の抽出 | 差分ゼロなら終了(記録のみ) |
| 7 | 有効日検証 | 有効開始日の妥当性(過去遡及は要人間確認) | 保留+人間レビュー |
| 8 | 廃止日検証 | 廃止レコードの参照残存確認 | 保留 |
| 9 | 経過措置検証 | 経過措置期間・移行先コードの整合 | 保留+人間レビュー |
| 10 | コード重複検証 | 主キー・コード体系内の重複 | 中止 |
| 11 | 参照整合性検証 | 他マスター・CodeMappingRegistry との参照整合 | 保留+CODE_MAPPING_REVIEW_REQUIRED |
| 12 | 算定エンジン回帰テスト | golden test 一式(算定実装後) | **本番反映禁止**・保留 |
| 13 | レセプト出力回帰テスト | 記録条件バリデーション一式(実装後) | 同上 |
| 14 | 帳票回帰テスト | 帳票表示・薬価表記の回帰 | 同上 |
| 15 | 医療安全影響チェック | 名称類似・規格変更・削除薬剤の影響一覧 | 影響ありは人間レビュー必須 |
| 16 | 法令適合性影響チェック | 改定・通知との対応(version_watchlist 照合) | 同上 |
| 17 | 体験品質影響チェック | 検索インデックス再構築・表示影響 | 記録 |
| 18 | 影響レポート生成 | 差分・影響・テスト結果の要約(承認者向け) | — |
| 19 | ステージング反映 | ステージング環境へ適用 | 失敗は保留 |
| 20 | 承認 | 人間承認(承認者ロール【要確認】)。請求影響ありは必須、なしは自動承認可(閾値は Phase 1 で確定) | 却下は破棄+記録 |
| 21 | 本番反映 | Cloud Core 本番へ版として追加(旧版は削除しない — 版共存) | ロールバックポイントへ復帰 |
| 22 | Edge Node 配布 | 署名付き配布・薬局別適用状況の追跡・版不一致検出 | 未達 Edge は旧版継続+警告表示 |
| 23 | ロールバックポイント作成 | 反映前状態の復元点 | — |
| 24 | 監査ログ保存 | 全段階の実行者・時刻・結果・ハッシュ | 欠落は incident |

## 4. 版選択ルール

- 算定・請求は「処方日・調剤日・請求月に対応する当時有効な版」を明示入力で使用する(算定エンジンの純粋関数制約 — CAL-001)。
- 版は `master_name + version + valid_from + valid_to(+経過措置)` で識別し、calculation_trace の masterVersions に必ず記録する(@yrese/trace 実装済み)。
- Edge Node は最終同期日時・最終マスター更新日時を常時表示する(offline_mode_matrix #23)。

## 5. 禁止事項

- 検証未通過マスターの本番反映
- 旧版マスターの削除(当時有効版の再現が請求・監査に必要)
- LOCAL_ONLY 中の Edge でのマスター更新適用(offline_mode_matrix #22)
- 配布元不明・ハッシュ不一致マスターの使用
