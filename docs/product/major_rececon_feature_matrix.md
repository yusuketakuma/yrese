## 1. 14分類×主要機能×MVP反映方針

| # | 分類 | 主要機能 | MVP反映方針 |
|---|---|---|---|
| 1 | 受付・患者管理 | 患者登録、保険・公費、資格確認、受付一覧、処方箋受付 | **MVP必須**(PRD-001 済み) |
| 2 | 処方入力 | 手入力、**前回Do**、QR/2次元シンボル、**OCR**、電子処方箋取込 | **MVP必須**(前回Do・OCR受け口は PRD-001 未記載 → 改版提案) |
| 3 | 算定・会計 | 調剤報酬点数、一部負担金、保険請求金額、公費、選定療養、返金・差額 | **MVP必須**(PRD-001 済み。選定療養は別建て計算 WP-0028) |
| 4 | レセプト請求 | 電子レセプト出力、請求前点検、月次締め、返戻再請求 | **MVP必須**(PRD-001 済み) |
| 5 | 帳票 | 領収証、調剤明細書、調剤録、薬袋、薬情、施設請求書 | **MVP必須**(施設請求書は PRD-001 未記載 → 改版提案・要人間判断) |
| 6 | 電子薬歴連携 | 薬歴一体型または密連携、**薬歴未記載チェック**、服薬指導支援 | **MVPではAPI連携必須**(薬歴未記載チェック=薬学管理料整合の請求前点検 → 改版提案) |
| 7 | 監査・安全 | 重複投薬、相互作用、OTC、疾患、アレルギー、処方監査 | **MVPでは外部監査API+最低限チェック**(自機実装はSaMDグレー = REG-005 で実装凍結中。外部監査システム結果の未加工転記+出典明示のみ) |
| 8 | オンライン資格確認 | 保険資格取込、薬剤情報、特定健診情報、請求前資格確認 | **MVP必須**(PRD-001 済み。ONS 待ち RB 継続) |
| 9 | 電子処方箋 | 受付、処方情報取得、調剤結果登録、送信、チェック結果 | **MVP必須。ただし公式IF確認前は停止**(既存 BLOCKED 維持) |
| 10 | 在庫管理 | 現在庫表示、自動発注、在庫連動、廃棄・配送最適化 | **MVPでは連携API口+現在庫表示を優先**(自動発注等は後続 → 改版提案) |
| 11 | 本部・多店舗 | 本部入力、チェーン集計、店舗横断管理、遠隔入力 | MVPでは設計のみ、Phase 2以降(PRD-002 済み) |
| 12 | BCP・オフライン | ローカル稼働、クラウド同期、サブサーバー、バックアップ | **MVP必須**(PRD-001 済み — Edge Node/LOCAL_ONLY/RECOVERY_SYNC) |
| 13 | UX・操作性 | サクサク、1画面、ファンクションキー、直感操作、教育コスト低減 | **MVP必須**(**二重UX=ガイド付き+ショートカット高速入力**は UIX 改版提案) |
| 14 | サポート・運用 | リモート支援、自動更新、マスター配信、障害対応 | **MVP必須**(PRD-001/OPS 済み。リモート診断の明記は改版提案) |

| 会計 | 領収、未収、返金、差額精算、会計履歴 |
| 帳票 | 領収証、調剤明細書、調剤録、薬袋、薬情、請求前点検リスト |
| レセプト | 電子レセプト生成、記録条件仕様バリデーション、月次締め、請求データロック |
| 請求前点検 | 入力漏れ、算定根拠、**薬歴未記載**、資格確認、公費、レセプト形式 |
| 電子薬歴連携 | API連携、薬歴未記載チェック、薬学管理料との整合 |

| マスター | 医薬品、薬価、調剤行為、コメント、公費、保険者、コードマッピング |
| オフライン | LOCAL_ONLY、仮算定、仮帳票、復旧後再検証 |
| UX | 1画面で見える、高速入力、ショートカット、ガイド、状態表示 |

## 4. PRD-001 改版提案(WP-0019 — 本書では改版しない)

既存 PRD-001(mvp_scope)との差分として、以下の追加・明確化を提案する:

1. **前回Do入力**(処方入力領域へ追加)
2. **OCR受け口**(入力経路の統一フロー: 手入力/QR/OCR/電子処方箋/前回Do → 同一 CanonicalPrescription に正規化。QR/OCRは原本扱いしない=既存原則の踏襲)
3. **電子薬歴連携API**(M12 連携の具体化: 薬歴未記載チェックイベント+薬学管理料整合)
4. **処方監査システム双方向API**(監査結果取込 — 未加工転記+出典明示。自機チェック実装は REG-005 凍結継続)
5. **在庫連携API口+現在庫表示**(処方入力画面での現在庫表示を後続 easy-win として設計)
6. **請求前点検の点検項目の明文化**(入力漏れ/算定根拠/薬歴未記載/資格確認/公費/レセプト形式)
7. **二重UX**(初心者ガイド+熟練者ショートカット)— UIX-002/004 と連動改版
8. **リモート診断**(OPS-004 と整合させ MVP 運用要件へ明記)
9. **施設請求書**(帳票領域への追加は施設調剤の MVP 範囲判断が必要 → **人間レビュー論点**)
10. 会計領域に**会計履歴**を明記
